Overview

What this website is

This repository powers Palmer Gill's personal project website. The public root page is a project launcher, and each project lives in its own static folder. Shared API features run through a FastAPI backend in backend/.

Frontend Static HTML, CSS, and JavaScript

Each app folder contains its own browser entrypoint and assets.

Backend FastAPI

Shared routes support stock data, poker, Bitcoin chat, and admin logs.

Hosting Vercel + Railway

Static routes stay on the website host; /api/* rewrites to Railway.

Design System

Visual language and components

The project launcher, apps, and docs share an editorial dark theme. Inter carries body and UI copy, Georgia handles display titles on hero sections, and SFMono-Regular renders code and tokens. The palette grounds on a near-black navy and pairs warm gold and bone with cooler blue, teal, and plum accents. Individual apps shift the accent — Stock Research leans blue, Bitcoin Chat leans orange, the project launcher and casino lean gold and warm bone.

Local Development

Run the site locally

Use the repo-level startup script to run the FastAPI backend and locally mounted static pages together.

./start.sh

Then open:

http://127.0.0.1:8000

Protected local app routes, FastAPI docs/OpenAPI JSON, and protected API routes require Basic Auth. Stock and Bitcoin app/API routes run in demo mode without credentials and use live providers with valid credentials. Poker, craps, blackjack, and /api/poker/* stay public. 

APP_AUTH_USERNAME=palmer APP_AUTH_PASSWORD=your-password ./start.sh

Backend logs are written to:

logs/backend.log

The shared backend reserves the local /docs path for generated FastAPI API docs. The static website docs page is served by production/static hosting at /docs/.

In local development, LOCAL_SITE_ROOT=true lets FastAPI serve the root site and active app folders from the same process. In production, the public website is static and the API backend is treated as a separate service.

Deployment

Production model

  • Static hosting serves the repo root and active app directories.
  • Vercel rewrites /api/* to the Railway backend.
  • Railway builds from the root Dockerfile and runs backend/app/main.py.

The Railway runtime command is:

uvicorn app.main:app --host 0.0.0.0 --port ${PORT:-8000}

Primary deployment variables:

APP_AUTH_USERNAME APP_AUTH_PASSWORD ALLOWED_ORIGINS DATABASE_URL USE_REAL_DATA ALLOW_MOCK_FALLBACK POLYGON_API_KEY FINNHUB_API_KEY BITCOIN_DATA_PROVIDER BITCOIN_RPC_URL BITCOIN_RPC_USER BITCOIN_RPC_PASSWORD BITCOIN_MEMPOOL_API_URL BITCOIN_MEMPOOL_TIMEOUT_SECONDS BITCOIN_CHAT_MODEL_TIMEOUT_SECONDS OPENAI_API_KEY

Authentication

Protected and public routes

App authentication supports HTTP Basic Auth and the signed HttpOnly session cookie created by /login/session. If APP_AUTH_PASSWORD is missing in production, protected routes return 503 instead of accidentally publishing private tools. Vercel middleware skips that fail-closed behavior outside Vercel so local static development is not blocked. Stock and Bitcoin routes allow unauthenticated demo access, while valid Basic Auth credentials unlock live provider-backed data.

Public

  • /
  • /docs/*

  • /bitcoin-chat/* demo mode
  • /poker/*

Single-player blackjack with a six-deck shoe, dealer hits soft 17, 3:2 naturals, double after split, pair splitting, split aces, insurance, and focused game-rule tests.

Chat interface connected to mempool.space by default, with an optional read-only Bitcoin Core RPC provider through Railway and a Cloudflare Tunnel. It can answer questions about blocks, transactions, mempool fees, node status, and mined-BTC data.

Data Providers

External services

Polygon.io

Primary Stock Research data source.

  • Ticker search: /v3/reference/tickers
  • Ticker details: /v3/reference/tickers/{ticker}
  • Previous close: /v2/aggs/ticker/{ticker}/prev
  • Historical prices: /v2/aggs/ticker/{ticker}/range/*
  • Financial statements: /vX/reference/financials

Bitcoin data

mempool.space is the default live provider for Bitcoin Chat. A private read-only Bitcoin Core RPC node remains available as an optional provider.

  • Default provider: BITCOIN_DATA_PROVIDER=mempool
  • Production tunnel hostname: bitcoin-rpc.palmergill.com
  • Allowed methods include block, transaction, mempool, and fee-estimate reads.
  • Wallet, signing, broadcast, and private-key methods should stay disabled.
